An Enhanced Object-Oriented Programming-Based Web Page Linker

Python Requests Beautiful Soup OOP Encapsulation AI Class Generation

01 // Abstract

"In the realm of web scraping and data extraction, the conventional approach involves repeated utilization of libraries like Requests and Beautiful Soup to extract information from web pages. This often results in cluttered and inefficient code, as programmers are compelled to replicate the same operations for different web URLs."

"To address this challenge, we propose an innovative solution: an Object-Oriented Programming (OOP) based web page linker, which encapsulates the functionality of web pages into reusable Python modules or libraries. The traditional approach for this concept includes the extraction of the input data source from the page and manipulating it as per user convenience. This technique is hectic, time-inefficient, complex, and verbose."

"Our approach in regards to improving the traditional for the same is the AI extracts the particular division (<div>) which is responsible for executing certain functions and creates that functionality into a class. Many times the techniques of input and execution vary from web page to web page which increases the workload on the AI as the techniques are unique to each web page. As a result, we have developed a project that addresses web-page complexity and increases the flexibility for the user to use a certain web page moreover it provides an efficient approach that creates a streamlined and time-efficient experience."

02 // Core Methodology & Architecture

Traditional scrapers are highly coupled to the DOM structure, breaking whenever structural classes or layouts change. Our architecture shifts this complexity onto an AI-driven encapsulation model:

  • AI-Driven Division (⟨div⟩) Extraction: Instead of manual DOM mapping, the system uses localized AI models to identify the specific DOM divisions (<div>) executing key page functionalities.
  • Dynamic Class Encapsulation: The identified division structure is programmatically encapsulated into self-contained Python class modules, exposing high-level functional APIs (e.g., page.get_pricing()) rather than exposing raw BeautifulSoup tags.
  • Input/Execution Adaptability: Since web structures vary, the system automatically translates unique page-specific input mechanisms (forms, buttons, inputs) into class methods, insulating the developer from web page complexity.
  • Reusable Python Modules: Compiled schemas are cached as standard local Python libraries, enabling immediate re-use across web scraping pipelines.

03 // Key Contributions

  • DOM-Agnostic Scrapers: Shields user scripts from direct DOM complexity by delegating structural changes to encapsulated AI class updates.
  • Streamlined Development: Replaces hectic, verbose, and duplicate Requests/BeautifulSoup scripts with high-level Python object models.
  • Flexible Web Automation: Extends user flexibility by supporting dynamic elements and handling page-specific input actions automatically.
